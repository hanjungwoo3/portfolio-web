## 1. 시장 시간 + sleeping 처리

### 1.1 시장별 거래 시간

| 시장 | 시간 (현지) | 시간대 |
|------|------------|--------|
| **KR** (한국) | 09:00 ~ 15:30 | Asia/Seoul (정규장만) |
| **US** (미국 주식) | 04:00 ~ 20:00 | America/New_York (PRE+정규+POST) |
| **US_INDEX** (지수) | 09:30 ~ 16:00 | America/New_York (정규장만) |
| **JP** (일본) | 08:30 ~ 15:30 | Asia/Tokyo |
| **OTHER** (환율/선물) | 항상 열림 | — |

주말 (토/일) 자동 휴장.

### 1.2 한국 장 세션 phase (`krSessionPhase`)

| phase | 시간 (KST) | 의미 |
|-------|-----------|------|
| **REGULAR** | 09:00 ~ 15:30 | 정규장 (실시간 변동) |
| **EXTENDED** | 08:00~08:50, 15:30~20:00 | 동시호가 / 시간외 단일가 |
| **CLOSED** | 그 외 모든 시간 | 장 종료 |

### 1.3 보유 종목 sleeping 판정 (`isHoldingSleeping`)

| phase | 활성 / 휴면 판정 |
|-------|-----------------|
| **REGULAR** | 항상 활성 (sleeping = false) |
| **CLOSED** | 항상 휴면 (sleeping = true) |
| **EXTENDED** | 마지막 체결 시각으로부터 **10분 경과** 시 휴면 |

→ EXTENDED 시간에 거래 활발하면 카드 활성, 거래 끊기면 자동 흐려짐.

### 1.4 Fade out 동작

sleeping 상태가 되면:
- 카드 전체 **opacity 60%** (60% 투명) — 회색조로 흐려짐
- 종목명 옆에 **`zzz`** 마크 표시
- transition-opacity로 부드럽게 전환

### 1.5 어제 대비 표시는 유지

- 장마감 후에도 `(price - base) / base × 100` = "어제보다 +X%" 그대로 표시
- 다음 장 시작 직전까지 유효 (당일 종가 vs 전일 종가)
- 변동 0이면 "어제보다" 행 자체 숨김

## 2. 색상 체계 (가격/손익)

### 2.1 한국식 컨벤션

| 부호 | 색 | 의미 |
|------|----|------|
| **+** (상승/수익) | 🔴 빨강 (`text-rose-600/700`) | 좋은 신호 |
| **−** (하락/손실) | 🔵 파랑 (`text-blue-600/700`) | 주의 신호 |
| **0** (변동 없음) | ⚪ 회색 (`text-gray-500`) | 중립 |

→ 미국식과 반대. 데스크톱 v1/v2와 동일.

### 2.2 카드 배경색 (보유 종목 손익)

| 손익 | 배경 + 테두리 | 의미 |
|------|--------------|------|
| pnl > 0 (수익) | `bg-rose-50/70 border-rose-200` | 옅은 빨강 |
| pnl < 0 (손실) | `bg-blue-50/60 border-blue-200` | 옅은 파랑 |
| pnl = 0 또는 미보유 | `bg-white border-gray-200` | 흰색 |

### 2.3 가격 / 어제보다 / 전체수익 텍스트

- 모두 부호에 따라 빨강/파랑 자동 적용 (`signColor()` 함수)
- "어제보다" 변동값은 **bold + text-base** 강조
- "전체수익" 금액은 일반, **% 만 bold**

종목명 옆 작은 뱃지 + 종목명 pill 배경색이 함께 변경됩니다.

### 3.1 뱃지 종류

| 뱃지 | 뱃지 색 | pill 배경 | 의미 |
|------|--------|-----------|------|
| **위험** | 🟥 빨강 진한 (`bg-red-700`) | `bg-rose-200` 옅은 빨강 | 시장 위험 경고 |
| **관리** | 🟥 빨강 진한 | 옅은 빨강 | 관리 종목 (상장폐지 위험) |
| **정지** | ⚫ 회색 (`bg-gray-500`) | `bg-gray-300` 옅은 회색 | 거래 정지 |
| **경고** | 🟧 주황 (`bg-orange-600`) | `bg-orange-200` 옅은 주황 | 일반 경고 |
| **과열** | 🟧 주황 | 옅은 주황 | 과열 종목 |
| **환기** | 🟧 주황 | 옅은 주황 | 투자 환기 |
| **주의** | 🟨 앰버 (`bg-amber-500`) | `bg-amber-200` 옅은 앰버 | 투자 주의 |

- 데이터 소스의 `warning` 필드 값에 따라 자동 매핑
- pill 배경이 바뀌면 한눈에 종목 위험도 식별 가능

## 4. 임계값 (9% 손절·트레일링)

데스크톱 v2와 동일한 **−9% 임계값** 두 가지.

### 4.1 손절 (Stop Loss)

| 조건 | 시각 효과 |
|------|----------|
| 보유 종목 + `(현재가 × 0.998 − 매수가) / 매수가 × 100 ≤ −9%` | 전체수익 % 에 **빨간 배경 + 흰 글씨** (`bg-rose-600 text-white`) |

→ 수수료 차감(0.2%) 후 실수익 기준.

### 4.2 트레일링 스탑 (Peak Drop)

| 조건 | 시각 효과 |
|------|----------|
| 피크가 매수가 위로 오른 적 있고, **(현재가 − 피크) / 피크 × 100 ≤ −9%** | 피크 % 에 **파란 배경 + 흰 글씨** (`bg-blue-600 text-white`) |

→ 즉 "최고점 대비 9% 이상 하락" 알림. 매수가가 아닌 **피크 기준**.

### 4.3 9% 넘을 때 동작 정리

| 상황 | 표시 |
|------|------|
| 매수가 대비 −9% 이하 | 전체수익 `(−9.50%)` 빨간 배경 + 흰 글씨 |
| 피크 대비 −9% 이하 | 피크 `(−12.30%)` 파란 배경 + 흰 글씨 |
| 둘 다 해당 | 두 표시 모두 적용 |
| −9% 미만 (−5% 등) | 일반 빨강/파랑 텍스트만 (배경 강조 없음) |

## 5. 틱 화살표 (직전 체결 대비)

가격 좌측에 표시되는 작은 화살표 — 데스크톱 v2와 동일 패턴.

| 직전 대비 | 첫 전환 | 연속 |
|-----------|---------|------|
| 상승 | **▵** (속빈 위쪽 삼각형) | **▲** (속찬 위쪽 삼각형) |
| 하락 | **▽** (속빈 아래쪽 삼각형) | **▼** (속찬 아래쪽 삼각형) |
| 동일 | 직전 화살표 유지 | 유지 |

색상: 상승 = 빨강 (`text-rose-600`), 하락 = 파랑 (`text-blue-600`).

## 6. 수수료 / 손익 계산

### 6.1 매도 수수료

```
SELL_FEE_PCT = 0.2%
실수익 단가 = 현재가 × (1 − 0.002) = 현재가 × 0.998
```

→ "전체수익" 표시는 항상 수수료 차감 후 실수익 기준 (낙관 편향 방지).

### 6.2 손익 공식

```
pnl = round((현재가 × 0.998 − 매수가) × 보유수량)
pnl% = ((현재가 × 0.998 − 매수가) / 매수가) × 100
```

### 6.3 어제 대비

```
dayDiff = 현재가 − base (전일 종가)
dayPct  = dayDiff / base × 100
```

→ 수수료 미반영 (단순 가격 변동).

카드 우측 그리드 — 12개 행 (외국인보유율 + 11개 매매 주체).

### 7.1 강조 행

| 행 | 폰트 |
|----|------|
| **외국인** | text-xs **font-bold** (가장 큰 강조) |
| **기관** | text-xs **font-bold** |
| **연기금** | text-[11px] **font-medium** |
| 그 외 9개 | text-[11px] 일반 |

### 7.2 강조 행 색상

| 매매 부호 | 라벨/값 색 | 행 배경 |
|----------|-----------|---------|
| 양수 (순매수) | text-rose-700 (진한 빨강) | bg-rose-50 (옅은 빨강) |
| 음수 (순매도) | text-blue-800 (진한 파랑) | bg-blue-50 (옅은 파랑) |
| 0 또는 데이터 없음 | text-gray-500 | (배경 없음) |

### 7.3 일반 행 (9개 — 개인/금융투자/투신/사모/보험/은행/기타금융/기타법인)

- 라벨: `text-gray-600`
- 값: 부호 색 (signColor) 적용
- 데이터 없음: `text-gray-400` 으로 "−" 표시

### 7.4 외국인보유율 (특수)

- 데이터 없거나 0이면 셀 전체 숨김 (그리드 자리는 유지)
- 표시 시 `XX.XX%` 형식, 회색

## 8. 프록시 인프라

### 8.1 공개 4-way 프록시 (기본)

전 사용자가 공유하는 무료 CORS 프록시 4개 — 라운드 로빈 + 자동 fallback.

| 프록시 | 무료 한도 | 위치 |
|--------|----------|------|
| Cloudflare Worker | 100k req/일 | `*.workers.dev` |
| Vercel Edge Function | 100k req/일 | `*.vercel.app` |
| Deno Deploy | 100k req/일 | `*.deno.net` |
| Render Web Service | 750h/월 (무한 req) | `*.onrender.com` |

**합계 약 40만 req/일** — 사용자 늘면 한도 초과 위험.

### 8.2 전용 프록시 (권장)

본인만 사용하는 Cloudflare Worker. → [`workers/proxy/DEPLOY-USER.md`](workers/proxy/DEPLOY-USER.md)

- 본인 100k req/일 전용
- 폴링 주기 5/10/30/60초 자유 선택
- 무료, 약 10분 소요, 코딩 지식 불필요

### 8.3 헬스체크 + healthy-first 정렬

- 각 프록시 호출 결과(성공/실패)를 클라이언트가 추적 (`src/lib/proxyStatus.ts`)
- 30초 윈도우 내 5회 연속 실패 → "다운"으로 마킹
- 매 fetch 시 healthy-first 순서로 재정렬
- 다운된 프록시도 N분 후 자동 재시도

### 8.4 Adaptive Polling

다운 프록시 수에 따라 갱신 주기 자동 증가:

```
실제 갱신 = base × (1 + 다운 카운트)
```

| 다운 수 | 5초 base | 10초 base |
|---------|---------|----------|
| 0 | 5초 | 10초 |
| 1 | 10초 | 20초 |
| 2 | 15초 | 30초 |
| 전부 | 갱신 중지 | 갱신 중지 |

## 9. 사용자 설정 (localStorage)

### 9.1 저장 키

| 키 | 의미 | 기본값 |
|----|------|--------|
| `portfolio_personal_proxy_url` | 전용 프록시 URL | 없음 (공개 4-way 사용) |

### 9.2 데이터 보관 정책

- 모든 데이터 **브라우저 로컬 저장** (IndexedDB + localStorage)
- 외부 서버 전송 0 (앱 자체가 GitHub Pages 정적 호스팅)
- 디바이스/브라우저별 분리 — 동기화 없음
- 보유 종목 (`holdings`) / 피크 (`peaks`)는 IndexedDB
- 설정 (`프록시 URL` / `폴링`)은 localStorage

### 10.6 StockCard

- **가격 박스**: 종목명 pill + 위험 뱃지 + 고가/현재가/저가 + 거래량
- **통계 박스**: 섹터 + 매수가 + 피크 + 어제보다 + 전체수익 + 목표
- **투자자 박스**: 12개 매매 주체 + 외국인보유율

## 11. 그룹 시스템 (보유 / 사용자 그룹)

### 11.1 데이터 구조

각 종목은 `account` 필드로 그룹 구분:

```json
{ "ticker": "005930", "name": "삼성전자",
  "shares": 10, "avg_price": 70000,
  "account": "퇴직연금" }   // ← 이 필드가 그룹 키
```

- `account: ""` (빈 문자열) → "**보유**" 탭
- `account: "퇴직연금"` → "**퇴직연금**" 탭 (사용자 자유 명명)
- 같은 종목이 **여러 그룹에 동시 존재 가능** (각 그룹마다 별도 row)
  → 예: 삼성전자를 "보유"에도 5주, "퇴직연금"에도 3주 보유 가능

### 11.2 탭 표시 순서

| 순위 | 탭 | 설명 |
|------|-----|------|
| 1 | **📈 미국 증시** | 시스템 탭 — 환율/지수/섹터별 ETF (UsMarketTab) |
| 2 | **💼 보유** | `account=""` 종목 (기본 그룹) |
| 3+ | **🏷 사용자 그룹** | 알파벳 순 (퇴직연금/관심/장기투자 등 자유) |

### 11.3 시스템 reserved 이름 (사용 불가)

| 예약 이름 | 용도 |
|----------|------|
| `""` (빈 문자열) | "보유" 탭 — 기본 그룹 |
| `관심ETF` | 미국증시 탭의 섹터별 ETF 컬럼에서만 사용 (탭 안 만듦) |
| `__us-market__` | 시스템 미국증시 탭 키 (내부) |

### 11.4 그룹 추가 방법

**방법 1 — 검색 다이얼로그 (가장 빠름)**

1. 헤더 **🔍 검색** 클릭
2. 종목 검색 후 결과 행 체크
3. 하단 "**그룹 적용:**" 영역에서:
   - 기존 그룹 버튼 클릭 → 마킹 (✓ 표시)
   - 또는 "**새 그룹**" 입력란에 이름 입력 → 즉시 마킹
4. 수량 입력 + **[일괄적용]** → "보유"에도 + 마킹된 그룹들에도 함께 추가

**방법 2 — 종목 수정**

1. 카드 hover → **✏️** 버튼
2. EditHoldingDialog 에서 그룹(account) 변경
3. 저장

**방법 3 — JSON import**

⚙️ 설정 → JSON 붙여넣기 시 각 종목의 `account` 필드 그대로 import.

### 11.5 그룹 관리 (Tabs hover)

| 동작 | 방법 |
|------|------|
| **이름 변경** | 탭 hover → **✏️** → prompt 새 이름 → 그 그룹 종목 일괄 account 변경 |
| **그룹 삭제** | 탭 hover → **🗑** → confirm → 그 그룹 종목 모두 삭제 (되돌릴 수 없음) |
| **종목 1개만 그룹에서 제거** | 카드 hover → **✕** → confirm (그 그룹에서만 제거 — 다른 그룹의 같은 종목은 유지) |

### 11.6 검색 다이얼로그의 ✓그룹 배지

종목명 옆 `✓퇴직연금` 같은 작은 배지 = "이 종목이 그 그룹에 이미 있음" 표시.
**클릭하면 그 그룹에서 그 종목만 제거** (즉시).

## 16. 부록 — DevTools Console 진단

브라우저 F12 → Console 탭에 붙여넣어 사용:

### 현재 설정 확인
```js
console.table({
  '전용 프록시': localStorage.getItem('portfolio_personal_proxy_url') || '(없음)',
  '폴링 주기': (Number(localStorage.getItem('portfolio_personal_poll_ms') || 10000)/1000) + '초'
});
```

### 공개 4-way 헬스체크
```js
const PROXIES = [
  'https://portfolio-proxy.hanjungwoo.workers.dev',
  'https://vercel-proxy-chi-tawny.vercel.app',
  'https://epic-zebra-73.hanjungwoo3.deno.net',
  'https://portfolio-render-proxy.onrender.com',
];
const TEST = 'https://query1.finance.yahoo.com/v8/finance/chart/AAPL';
await Promise.all(PROXIES.map(async p => {
  const t0 = performance.now();
  try {
    const r = await fetch(`${p}/?url=${encodeURIComponent(TEST)}`);
    return { proxy: p.replace('https://',''), status: r.status, ms: Math.round(performance.now()-t0) };
  } catch {
    return { proxy: p.replace('https://',''), status: 'ERROR', ms: Math.round(performance.now()-t0) };
  }
})).then(r => console.table(r));
```

### 전용 ↔ 공개 전환
```js
// 전용으로 전환
localStorage.setItem('portfolio_personal_proxy_url', 'https://your-proxy.workers.dev');
location.reload();

// 공개로 복귀
localStorage.removeItem('portfolio_personal_proxy_url');
location.reload();
```

### 폴링 주기 강제 변경
```js
localStorage.setItem('portfolio_personal_poll_ms', '5000');  // 5초
location.reload();
```
